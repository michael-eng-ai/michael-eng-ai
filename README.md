Sou um **Engenheiro de Dados Sênior** com forte paixão por desenhar, construir e otimizar plataformas de Big Data modulares e escaláveis. Minha missão é ajudar as empresas a extrairem o máximo de valor estratégico dos seus dados, usando tanto as melhores práticas de Engenharia de Dados quanto integrando as mais recentes inovações em Inteligência Artificial Generativa (GenAI).

Com passagem por empresas como **Five Acts**, **Act Digital**, **Vert** e **ART IT**, tenho consolidado meu foco em migrações para Cloud (AWS, Azure, GCP), processos de ETL/ELT distribuídos com ferramentas modernas (Databricks, DBT, Apache Airflow) e streaming em tempo real (Kafka).

## 🏆 Projetos em Destaque no Portfólio

Aqui estão os principais projetos de arquitetura de dados que desenvolvi, provando expertise em Cloud e Engenharia de Dados Escalável:

### 🤖 1. [AI-Data-Analyst-Bot](https://github.com/michael-eng-ai/AI-Data-Analyst-Bot)
*Um assistente virtual híbrido utilizando arquiteturas de GenAI para análise de banco de dados e leitura de documentos.*
- **Arquitetura:** `Text-to-SQL` (Consultas diretas a relatórios) + `RAG` (Busca semântica avançada).
- **Tech Stack:** Python, Streamlit, Langchain, Google Gemini 1.5, ChromaDB, SQLite, Docker, Terraform (GCP), GitHub Actions (CI/CD).

### ☁️ 2. [aws-databricks-lakehouse](https://github.com/michael-eng-ai/aws-databricks-lakehouse)
*A fundação do Modern Data Lake: Arquitetura Medallion desacoplando Storage e Computação na nuvem da Amazon.*
- **Arquitetura:** Ingestão de logs em S3 (Raw) processados por clusters PySpark (Silver/Gold) operando sobre formato open-source Delta Lake.
- **Tech Stack:** AWS S3, IAM, Terraform, Databricks (PySpark), Delta Lake, Python (Boto3).

### 📊 3. [gcp-dbt-modern-data-stack](https://github.com/michael-eng-ai/gcp-dbt-modern-data-stack)
*O fluxo estado-da-arte em Analytics Engineering. Transformação ágil e testada de modelagens dimensionais.*
- **Arquitetura:** Orquestração automatizada (Serverless CRON) executando extrações vis API, carga em Storage/Analytics e testes nativos.
- **Tech Stack:** Google Cloud Platform (GCS, BigQuery), dbt Core, Python, Terraform, GitHub Actions.

### ❄️ 4. [azure-snowflake-pipeline](https://github.com/michael-eng-ai/azure-snowflake-pipeline)
*Pipeline "Zero-ETL" integrando nuvens distintas de forma passiva, automatizada e escalável.*
- **Arquitetura:** Conta Azure gerando o Storage Gen2 e Data Warehouse reagindo a eventos de ingestão nativamente mapeandos dados via Auto-Ingest.
- **Tech Stack:** Azure Data Lake Storage Gen2, Snowflake (Stage, Pipe, Streams), Terraform, SQL (DDL), Python.

### 🌪️ 5. [streaming-kafka-fastapi](https://github.com/michael-eng-ai/streaming-kafka-fastapi)
*Microsserviços Event-Driven lidando com altíssima vazão de dados contínuos (Data in Motion).*
- **Arquitetura:** Cluster conteinerizado conectando Produtor e Consumidor assíncronos que convertem e persistem "ticks" financeiros servidos velozmente via Web REST API.
- **Tech Stack:** Apache Kafka (Broker), Redis, FastAPI, Python, Docker (Compose).
